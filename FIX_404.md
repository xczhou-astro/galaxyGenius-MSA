# Fix 404 Error on GitHub Pages

## Problem
The website shows 404 because:
1. The HTML files are in `_build/html/` instead of the root
2. The `.nojekyll` file is missing (required for Sphinx)

## Solution

Run these commands:

```bash
# Make sure you're on gh-pages branch
git checkout gh-pages

# Copy HTML files to root
cp -r _build/html/* .

# Create .nojekyll file (critical!)
touch .nojekyll

# Add and commit
git add .
git commit -m "Fix: Move HTML files to root and add .nojekyll"

# Push to GitHub
git push origin gh-pages
```

## Verify GitHub Pages Settings

1. Go to repository → **Settings** → **Pages**
2. Make sure:
   - **Source**: "Deploy from a branch"
   - **Branch**: "gh-pages"
   - **Folder**: "/ (root)"

## Wait a Few Minutes

GitHub Pages can take 1-5 minutes to update after pushing.
